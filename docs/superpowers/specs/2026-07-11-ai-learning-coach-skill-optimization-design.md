# AI Learning Coach Skill 拆分优化设计

## 背景与目标

当前 `skills/ai-learning-coach/SKILL.md` 同时承担触发、路由与六种学习方法的详细流程，存在主文件偏重、元数据不规范、部分行为冲突及边界条件缺失的问题。

本次优化目标：

- 精简主文件，使其只负责触发、路由、通用交互约束与引用索引。
- 将六种学习方法拆为可独立加载和维护的参考文件。
- 修正费曼流程、资源真实性、短计划复习、评分量表和提醒声明等行为问题。
- 让学习计划成为可勾选、可记录完成日期、掌握程度和学习心得的 Markdown 计划。
- 增加零第三方依赖的静态校验与可配置的真实 Agent 行为测试。

### 职责边界

- `SKILL.md`：仅保留合法 frontmatter、适用/不适用条件、方法路由、最小必要澄清、通用约束和参考文件索引。
- `references/*.md`：每个文件只描述一种学习方法，统一包含“何时使用、必要输入、执行流程、输出契约、边界情况、禁止事项”。
- `validate_skill.py`：执行确定性的结构、元数据、引用和内容契约校验。
- `scenarios.json`：以机器可读方式描述输入、必须行为、禁止行为和可选数量约束。
- `run_behavior_tests.py`：通过可配置 Agent 命令执行场景，并输出通过、失败和人工复核提示。
- `tests/README.md`：说明运行方式、退出码、场景格式和新增回归测试的方法。

## 主文件设计

Frontmatter 只允许：

```yaml
---
name: ai-learning-coach
description: Use when 用户想学习新主题、制定学习计划、自测掌握程度、整理速查表、筛选学习资源，或验证自己是否真正理解某个概念。
---
```

description 只描述触发条件，不概述六种方法的执行流程。

路由表根据用户意图选择且只加载一个主要参考文件。意图不明确、且不同选择会显著影响结果时，一次只询问一个关键问题；用户信息已足够时直接执行。主文件不复制各方法的详细步骤。

通用约束：

- 互动式方法每轮只完成当前动作，然后等待用户回应，不模拟用户回答。
- 练习和完成标准必须可观察、可验证。
- 默认只推荐一个最相关的下一步。
- 复习时间默认是建议；只有自动化工具实际执行成功后，才能声称已创建提醒。
- 需要当前资源时必须核验；无法核验时明确说明限制，不编造具体信息。

## 六种方法设计

### 学习阶梯

- 用 2～3 个诊断问题定位起点；若用户已给出明确经验和目标，可声明假设后直接判断。
- 保持五层结构，但每层必须提供知识/技能、常见错误和可观察的晋级标准。
- 输出五层概览、当前层判断证据和到下一层的最近路径。

### 聚焦学习计划

- 将二八原则作为优先级启发，不承诺严格的 20% 内容覆盖 80% 用途。
- 按用户目标、高频使用、迁移价值和前置依赖选择核心内容。
- 默认每次学习不超过两小时；根据总时长和用户可用时间确定次数。
- 复习节点动态分配：不超过三次时在末次综合复习；四至六次在中点和终点复习；七次以上约每三次累计复习一次，且节点不得超过总次数。
- 输出概览、阶段、里程碑、时间分配、核心内容和逐次计划。

每次计划采用可持续维护的 Markdown 打卡格式：

```markdown
## 第 1 次学习：理解核心概念

- [ ] 完成知识学习：……
- [ ] 完成练习：……
- [ ] 回答复盘问题：……
- [ ] 将错误记入错误日志
- [ ] 完成本次学习

**预计用时：** 90 分钟
**完成日期：** ____年__月__日
**学习心得：**
> 在这里记录新理解、困难、错误原因或下一步调整。

**掌握程度：** ☐ 不熟悉　☐ 基本掌握　☐ 能独立应用
```

总览包含总体进度清单。普通学习保留简短心得区，阶段复习和重要里程碑使用更完整的复盘模板。不得替用户预先勾选完成状态。

### 苏格拉底测试

- 确认测试范围后，默认五道主问题，必要时最多增加两道诊断题。
- 每轮只问一道题并等待回答。
- 回答后用固定量表评分并给出证据：0～2 为核心概念错误，3～5 为部分事实正确但关系不清，6～7 为基本正确但有重要遗漏，8～9 为准确且能应用，10 为准确、完整且能迁移。
- 记录概念混淆、遗漏、记忆错误或应用错误；达到题数或已有充分诊断证据后结束。
- 最终输出错误模式、弱项加练和复习时间建议。

### 一页速查表

- 将“一页”定义为正文最多六个区块、约一千个中文字符，而非物理纸张页数。
- 包含本质定义、核心概念、真实例子、常见错误、使用前检查清单和快速自测。
- 自测问题与答案分开，避免用户未经检索就看到答案。
- 输出复习时间建议，但不声称已创建系统提醒。

### 资源筛选

- 推荐当前资源前必须核验，优先官方、原作者或权威来源。
- 每个资源说明适用对象、难度、使用方式、可跳过部分和核验日期。
- 禁止虚构标题、链接、章节、价格或可用状态。
- 无法联网或验证时，说明限制，并请求用户提供候选资源或允许检索；不得输出伪装成已核验的具体推荐。
- 在资源已核验后，用其设计七天路径。

### 费曼验证

- 验证模式必须由用户先讲，Agent 不预先给标准解释。
- 通过追问定位术语遮蔽、因果断裂、重要遗漏和无法举例的问题。
- 用户完成第二次讲解后，再提供对照解释和元认知复盘。
- 掌握标准是能用简单语言说明并给出自己的例子。
- 若用户只是要求 Agent 解释概念，应进入教学模式并说明这不是费曼验证，不要求用户先讲。

## 静态校验设计

`validate_skill.py` 使用 Python 标准库并返回非零退出码表示失败。校验内容：

- 主文件存在，frontmatter 只含 `name` 和 `description`。
- name 格式合法；description 以 `Use when` 开头且长度合理。
- 六个参考文件存在且被主文件引用。
- 每个参考文件含统一必需章节。
- Markdown 本地相对链接均能解析。
- 主文件不包含六种方法的详细执行步骤。
- 不出现费曼流程的已知冲突表达。
- 学习计划参考文件包含复选框、完成日期、掌握程度和学习心得契约。
- `scenarios.json` 是合法 JSON，场景 ID 唯一且字段完整。

## 行为测试设计

`scenarios.json` 至少覆盖：

- 模糊学习请求选择单一合适方法，不一次展开全部流程。
- 四小时计划不安排不存在的第六、第九次复习。
- 学习计划包含可勾选任务、完成日期、掌握程度和心得区。
- 苏格拉底测试首轮只问一道题并等待。
- 费曼验证让用户先讲且不提前泄露答案。
- 普通概念解释进入教学模式，不误判为费曼验证。
- 无法核验资源时说明限制且不虚构链接。
- 速查表遵守内容预算，并分开问题与答案。
- 仅输出复习日程时不声称提醒已创建。

行为测试由命令模板接入 `codex exec` 或其他 Agent。断言分两类：

- 硬断言：必需文本、禁止文本、正则表达式、问题数量和复选框等确定性规则。
- 人工复核：资源质量、解释准确度、难度是否合适等主观判断。

## 错误处理与退出码

- 静态校验：成功返回 `0`，校验失败返回 `1`。
- 行为测试：全部硬断言通过返回 `0`；场景失败返回 `1`；Agent 命令未配置返回 `2`。
- 单个场景超时或命令异常时记录失败并继续其他场景。
- Agent 返回空内容时作为场景失败处理。
- 报告保留必要的标准错误摘要，但不输出环境变量或完整敏感命令配置。
- 无网络条件下，资源场景验证 Agent 是否诚实声明无法核验，而不要求其强行给出推荐。

## 验收标准

- 主文件完成合法化和精简，六个方法均能按需定位到独立参考文件。
- 所有已确认的行为修正写入明确、无冲突的契约。
- 静态校验可在无第三方依赖环境运行并通过。
- 行为测试包含上述回归场景，可通过命令模板运行真实 Agent。
- 测试文档足以让维护者新增场景和理解失败结果。
- 不修改与本 skill 无关的仓库文件。
