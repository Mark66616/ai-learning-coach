# 学习资源筛选

## 何时使用

当用户需要当前可访问、可信且适合其目标的课程、文档、书籍、视频或练习资源时使用。

## 必要输入

- 学习目标、当前水平、偏好形式与可用时间。
- 语言、预算、地区、设备或无障碍限制。
- 用户已有候选资源；没有候选时，使用可用的联网检索或核验工具直接查找。

## 执行流程

1. 当前推荐必须通过联网检索或可用工具核验；工具可用时直接核验，不先询问许可。优先官方文档、原作者材料、权威机构与一手来源，再考虑高质量二手讲解。
2. 核验标题、作者或机构、链接可访问性、内容范围及影响选择的价格或状态。价格、可用性等易变信息写明核验日期。
3. 按目标匹配而非热度排序。每项标注：适用对象、难度、具体用法、可跳过部分、核验日期。
4. 去除功能重复的资源，解释保留理由和取舍。
5. 仅在核验完成后，用入门、练习、输出与复习串成默认 7 天路径；每天指明资源片段、行动和完成证据。用户期限不是 7 天时，保留同一结构并压缩或展开到实际期限，明确调整关系。用户明确“只要清单”或“不要计划”时，不展开路径。

## 输出契约

- 推荐清单使用可点击的直接链接，并附来源或机构。
- 每项均包含“适用对象 / 难度 / 用法 / 可跳过 / 核验日期”。
- 区分“已核验事实”和“基于目标的判断”；不确定项显式标注。
- 默认在清单后给 7 天路径或按用户实际期限调整后的等价路径；如无法核验，则不输出伪装成已核验的路径。
- 用户明确只要清单或不要计划时，只输出核验后的清单与精简使用顺序；结尾用一句话提供可选 7 天路径入口，不强制展开，也不用追问阻塞清单。

## 边界情况

- 仅在工具不可用、登录受限或用户明确禁止联网时说明限制；可请求用户提供候选链接，在核验前只提供筛选标准，不给伪装推荐。
- 付费墙、地区限制或登录后可见：说明实际可见范围，不推断完整内容。
- 资源版本冲突：优先目标所需的当前官方版本，并标注旧版差异。
- 没有单一资源覆盖目标：选最小互补组合，避免资源堆砌。

## 禁止事项

- 不虚构标题、链接、章节、价格、库存、免费状态或更新状态。
- 不仅凭记忆提供“当前推荐”，不把搜索摘要当已读正文。
- 不用资源数量制造价值，不推荐与用户约束冲突的内容。
- 不在核验前设计依赖具体资源章节的 7 天路径。
