# 学习阶梯

## 何时使用

当用户需要判断当前水平、看清下一步，或把宽泛主题拆成可晋级的学习路径时使用。

## 必要输入

- 学习主题与目标。
- 已有经验、近期作品或能证明掌握程度的行为。
- 可用时间及目标期限；缺失时可先给不含日期的路径。

## 执行流程

1. 先把目标收敛为一个可观察产出明确的可验证子主题；主题已足够具体时直接确认，不额外缩小。
2. 再检查该子主题的现有证据。证据不足时只问 2–3 个诊断问题，覆盖“能否解释、能否操作、能否迁移”；信息充分时明确声明所用假设并直接判断，不重复提问。
3. 建立五层结构：入门识别、基础理解、独立应用、综合迁移、熟练指导。主题特殊时可改层名，但必须保持五层递进。
4. 每层分别写明：应掌握的内容、常见错误、可观察且可验证的晋级标准。
5. 用用户的回答、作品或经历标注当前层证据；证据矛盾时按较低层判断，并指出待验证项。
6. 给出从当前层到下一层的最近路径，只安排满足晋级标准所需的最小学习与练习。

## 输出契约

- 输出五层阶梯表，每层含“掌握内容 / 常见错误 / 可观察晋级标准”。
- 单列“当前层判断”，写出结论、证据、假设与不确定性。
- 单列“最近路径”，包含下一目标、1–3 个行动及完成证据。
- 不用学习时长、课程数量代替能力证据。

## 边界情况

- 用户完全没有经验且可验证子主题已明确：定位为第一层，并给最小起步动作，不进行羞辱式诊断；主题仍宽泛时先收敛主题，不急于定级。
- 用户理论强但实践弱：按目标所需能力分别标注，最终层级取关键短板。
- 目标跨度过大：先缩小到一个可验证子主题，再建阶梯。
- 用户自述与样例冲突：优先采用可观察样例，并说明原因。

## 禁止事项

- 不超过 3 个诊断问题，不把提问变成考试。
- 不输出含糊标准，如“基本了解”“感觉熟练”。
- 不跳过当前层证据，也不一次规划完整远期课程替代最近路径。
- 不把五层机械等同于固定课时、证书或年限。
