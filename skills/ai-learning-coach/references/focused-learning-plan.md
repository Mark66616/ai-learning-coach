# 聚焦学习计划

## 何时使用

当用户已有明确主题、期限或总学习次数，需要把精力集中到高价值能力并形成可勾选计划时使用。

## 必要输入

- 学习目标、当前水平和期望产出。
- 可安排的总次数、每次时长、起止日期或节奏。
- 必学范围、已有材料与不能触碰的范围。

## 执行流程

1. 以目标产出反推关键能力。二八法则只作为排序启发，不把“20%”包装成精确事实；保留所有达标所必需的基础与安全内容。
2. 每次学习上限为 2 小时；内容过多时增加次数或缩小范围，不延长单次负荷。
3. 安排累计复习且不得超出用户总次数：总次数 ≤3 时在末次复习；4–6 次时在中点和终点复习；≥7 次时约每 3 次安排一次累计复习，并让最后一次承担终点复习。
4. 先给总览，再逐次展开。普通学习聚焦新知与应用；阶段复习必须更完整，覆盖累计回忆、混合练习、错误重做、薄弱点判断和下一阶段调整。
5. 每次均保留用户完成后的记录位，不代填结果。

## 输出契约

- 总览必须包含：阶段、里程碑、时间分配、总体进度。总体进度逐次列为 `- [ ] 第 N 次：里程碑`，由用户完成时勾选，不写会随执行失真的静态 `0 / N` 或百分比。
- 每次学习均包含以下未勾选 Markdown 项：
  - [ ] 知识：具体学习点
  - [ ] 练习：可验证任务
  - [ ] 复盘：回忆与总结提示
  - [ ] 错误日志：记录错误、原因与修正
  - [ ] 完成本次：本次达成证据
- 五个复选项之后保留三个非复选记录字段：`完成日期：____`、`掌握程度：____`、`学习心得：____`。
- 标明每次预计时长（≤2 小时）及对应里程碑；阶段复习标记为“累计复习”。

## 边界情况

- 用户只给总小时数：按每次不超过 2 小时拆分，并声明建议次数。
- 总次数不足以覆盖目标：先按现有预算给出缩小范围后的最低可行计划，再提示可选扩展；不以追问是否增加次数阻塞当前计划。
- 已掌握部分内容：以简短验证替代重复学习，把时间转给短板。
- 日期不明确：使用“第 1 次”等相对顺序，不虚构日历日期。

## 禁止事项

- 不预先勾选任何项目，不替用户填写完成日期、掌握程度或学习心得。
- 不把二八法则当硬性比例，不为了“高收益”删除必要基础。
- 不安排超过 2 小时的单次学习，不增加用户未同意的次数。
- 不让复习仅等于重读；必须有提取、纠错与迁移。
