# AI 辅助学习方法集

## 概述

本技能提供6个经过学习科学优化的 AI 辅助学习方法，用于系统化地学习任意主题。每个方法可独立触发，也可组合使用。方法集合覆盖了从入门到精通的完整学习链路：结构化进阶、聚焦关键内容、检索测试、知识压缩、资源筛选、理解深度验证。

每个方法在原始设计基础上融入了以下学习科学环节：
- 预习评估（定起点，避免无脑从零开始）
- 刻意练习（针对薄弱点而非重复舒适区）
- 错误日志（记录归类，定期回顾）
- 间隔重复复习（对抗遗忘曲线）
- 元认知复盘（反思学习方法本身）

## 使用方式

用户表达以下意图时触发对应方法：

| 意图关键词 | 触发方法 |
|---|---|
| 学习阶梯、学习路径、从新手到精通、拆成等级 | 方法1：学习阶梯 |
| 学习计划、二八、重点学习、多少小时 | 方法2：二八学习计划 |
| 考考我、自测、提问测试、苏格拉底 | 方法3：苏格拉底提问测试 |
| 速查表、总结一页、cheat sheet、压缩 | 方法4：一页速查表 |
| 推荐资源、学习资源、看什么、书单 | 方法5：资源筛选 |
| 费曼、讲给我听、验证理解、讲明白 | 方法6：费曼讲解验证 |

用户未明确指定方法时，询问其当前学习阶段后推荐：
- 刚开始想了解主题 → 推荐方法1（学习阶梯）或方法5（资源筛选）
- 已有基础想系统学 → 推荐方法2（二八学习计划）
- 刚学完想检验 → 推荐方法3（苏格拉底测试）或方法6（费曼讲解）
- 学完想固化记忆 → 推荐方法4（一页速查表）

## 方法1：学习阶梯

**目的**：把任意主题拆成5个学习等级，形成清晰进阶路径。

**触发输入**：学习主题名称。

**执行流程**：

1. 先做预习评估：用2-3个快速问题探测用户当前水平，确定从第几层开始（避免无脑从第1层起步）。
2. 将主题拆成5个等级：
   - 第1层 完全新手：能理解基本概念和术语
   - 第2层 入门：能完成简单练习
   - 第3层 进阶：能解决常见问题
   - 第4层 熟练：能独立完成小项目
   - 第5层 精通：能独立做项目并优化
3. 每一层输出三部分：
   - 应该掌握什么（具体知识点和技能）
   - 常见错误是什么（具体陷阱，不要泛泛而谈）
   - 达到什么标准才能进入下一层（可验证标准，如"能独立完成XX"）
4. 标注用户当前所在层级，给出从当前层到下一层的最近路径。

**输出格式**：5层概览表 + 每层详细说明 + 当前层建议。

## 方法2：二八学习计划

**目的**：在有限时间内聚焦最重要的20%内容，设计成多次学习计划。

**触发输入**：学习主题 + 计划小时数。

**执行流程**：

1. 先做预习评估：探测用户当前水平，确定计划起点。
2. 识别该主题最重要的20%内容（覆盖80%实际用途的核心知识）。
3. 将计划小时数折算为学习次数（默认每次2小时，包含练习+资源+复盘）。
4. 为每次学习设计：
   - 本次学习目标（具体，可验证）
   - 学习内容（聚焦20%核心）
   - 练习资源（具体题目/项目/工具，禁止"找相关练习"这类空话）
   - 复盘问题（3-5个，检验本次掌握程度）
5. 在第3、6、9次插入回顾环节，复习前面内容（间隔重复）。

**输出格式**：20%核心内容清单 + 学习计划表 + 每次详情。

## 方法3：苏格拉底提问测试

**目的**：一次一个问题，从易到难，检验掌握程度并针对性补强。

**触发输入**：刚学完的主题名称。

**执行流程**：

1. 确认测试范围（用户刚学完的具体主题）。
2. 一次只问一个问题，从简单到困难。
3. 用户回答后：
   - 打分（0-10分）
   - 指出哪里对
   - 指出哪里错
   - 指出哪里理解不完整
   - 只重新解释用户没掌握的部分（不要全盘重讲）
4. 记录每次错误到错误日志：错误内容 + 错误类型（概念混淆/遗漏/记忆错误/应用错误）。
5. 测试结束后：
   - 生成错误日志汇总，归类典型错误模式
   - 针对最薄弱点设计3-5道针对性加练题（刻意练习，专挑弱项）
6. 安排下次复习时间（间隔重复：1天后/3天后/7天后）。

**关键约束**：一次只问一个问题，等用户回答后再继续。禁止一次抛出多个问题。

## 方法4：一页速查表

**目的**：把一个主题压缩成一页可快速回顾的速查表，并安排间隔复习。

**触发输入**：刚学完的主题名称。

**执行流程**：

1. 将主题压缩为一页，包含：
   - 一句话定义（本质，不是字典式描述）
   - 最重要的3-5个概念（每个配一句话解释）
   - 2-3个真实例子（具体场景，不要抽象）
   - 常见错误（3-5个，具体陷阱）
   - 使用前检查清单（5-7个yes/no问题）
   - 5个快速自测问题（能30秒内回答的）
2. 生成间隔复习提醒：
   - 第1次复习：1天后
   - 第2次复习：3天后
   - 第3次复习：7天后
   - 第4次复习：21天后
3. 每次复习时只重测自测问题，答错的概念回到速查表重读。

**输出格式**：一页速查表（结构化）+ 复习时间表。

## 方法5：资源筛选

**目的**：筛选最值得看的5个学习资源，设计7天学习路径。

**触发输入**：学习主题。

**执行流程**：

1. 筛选5个最值得看的资源，每个说明：
   - 适合谁（前置知识要求）
   - 难度（入门/进阶/精通）
   - 该怎么用（看哪部分，怎么练）
   - 不要浪费时间看哪里（明确指出可跳过的部分）
2. 用这5个资源设计7天学习路径：
   - 每天用什么资源、学什么、练什么
   - 第7天安排综合练习或小测验
3. 资源选择原则：质量优先于数量，宁可少而精。

**输出格式**：5个资源详情表 + 7天路径表。

## 方法6：费曼讲解验证

**目的**：用费曼学习法验证用户是否真正理解，并做元认知复盘。

**触发输入**：要验证的概念名称。

**执行流程**：

1. AI 先用12岁孩子能听懂的话解释这个概念（建立基线）。
2. 请用户用自己的话把概念讲回去（讲给一个不懂的人听）。
3. 用户讲完后，AI 评估：
   - 哪里讲对了
   - 哪里漏了
   - 哪里混淆了
   - 哪里只是用了一个很高级但自己也没真懂的词（重点揪这类"假懂"）
4. 让用户重新讲一次，直到讲明白。
5. 讲明白后，做元认知复盘：
   - 这次理解卡在哪里？
   - 之前以为懂了其实没懂的是什么？
   - 下次学类似概念时该怎么调整方法？
6. 判定标准：用户能用简单的话讲清楚，且能举出一个自己的例子，才算真正掌握。

**关键约束**：必须让用户先讲，AI 不要急于纠正。揪"假大词"时要明确指出哪个词是用户没真懂的。

## 通用原则

- 所有方法都应先确认用户的具体主题和当前水平，不要假设。
- 练习资源必须具体（具体题目/项目/工具链接），禁止"找相关练习"这类空话。
- 复盘问题必须可验证，禁止"你理解了吗"这类无效问题。
- 每次方法执行后，主动提示用户下一步可以调用哪个方法，形成学习闭环：
  - 方法1结束后 → 提示可用方法2制定计划，或方法5找资源
  - 方法2结束后 → 提示可用方法3自测
  - 方法3结束后 → 提示可用方法4做速查表固化
  - 方法4结束后 → 提示按复习时间表回来，或用方法6深度验证
  - 方法5结束后 → 提示可用方法2制定详细计划
  - 方法6结束后 → 提示可用方法4做速查表
